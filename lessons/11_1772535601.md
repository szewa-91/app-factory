# Lesson Learned: Robust Offset-Based Log Tailing (Task 11)

## Technical Insights
1. **Efficient Log Streaming**: Implementing a "terminal-like" viewer using byte offsets (via the X-Log-Offset header) is significantly more efficient than re-sending the entire file or fetching a fixed number of lines.
2. **Handle File Rotation Gracefully**: When using offsets, the server-side logic must handle cases where the requested offset is larger than the current file size (e.g., after log rotation). Returning data from the beginning (offset 0) in such cases prevents the client from missing logs when a file is swapped.
3. **Client-Side Buffer Management**: To maintain browser performance during long sessions, the client-side state should limit the number of lines kept in memory (e.g., using .slice(-1000)).
4. **Tool-Specific Pitfalls**: When using write_file or replace in certain LLM agent environments, literal newlines in single-quoted strings (e.g., .split('\n')) might be improperly escaped or literalized, leading to syntax errors like 'Unterminated string constant'. Using explicit \n or backticks is safer for code generation.
