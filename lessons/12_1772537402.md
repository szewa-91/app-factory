# Lesson Learned: Building Robust Terminal-Like Log Viewers in Next.js

## Technical Insights
1. **Byte Offset Tailing**: For large log files, using byte offsets (via headers like `X-Log-Offset`) is more efficient than reading the entire file or fetching the last N lines repeatedly. It allows for true streaming/tailing without redundant data transfer.
2. **Modal Focus and UX**: When implementing log viewer modals, providing an `ESC` key listener and auto-scroll functionality (with a manual toggle) significantly improves the developer experience.
3. **Robust Fetching**: Handling `404` errors gracefully in the UI (e.g., "Log file not found yet") is crucial for tasks that haven't started executing yet, preventing confusing generic error messages.
