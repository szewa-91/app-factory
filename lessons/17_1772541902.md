# Lesson Learned: Documentation Sync for Resilient Systems

## Technical Insights

1. **Self-Healing Documentation**: Documenting self-healing mechanisms (like `PREVIOUS_FAILURE.log` injection and PID monitoring) is crucial for agents to understand that they have context-rich diagnostic data available on retries. This reduces time-to-fix by making the agent aware of its "memory" of previous failures.
2. **Git Safety Net as a Feature**: Explicitly documenting the automatic rollback mechanism on verification failure provides safety guarantees for system-level changes. This allows the factory to self-correct without manual intervention when a task introduces regressions.
3. **Vision Anchors**: Using a `VISION.md` file as an absolute anchor ensures that even as the codebase evolves autonomously, it remains true to its core architectural principles (locality, simplicity, resilience). Documentation should always reference these anchors to maintain long-term alignment.
4. **Bootstrapping with Testing**: Integrating testing frameworks (Vitest, Playwright) directly into the bootstrap script ensures that all new applications start with a "test-first" foundation, making them easier to maintain and verify autonomously.
