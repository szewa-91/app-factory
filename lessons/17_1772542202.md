# Lesson Learned: Documentation Sync as System Integrity

## Technical Insights

1.  **Surgical Documentation Updates**: Documentation in autonomous systems is not just for humans; it's also for agents. Ensuring `GEMINI.md` and `README.md` are synchronized prevents "knowledge drift" where future agents rely on outdated mandates.
2.  **Addressing Technical Debt in Docs**: Just as code requires refactoring, documentation can accumulate "debt" (duplications, broken numbering). Cleaning this during a sync routine improves the clarity of system mandates.
3.  **Mandatory Commit Policy**: In an autonomous factory, atomic git commits are crucial for the "Git Safety Net" to work. Without them, the scheduler cannot reliably roll back to a known good state.

## Architectural Patterns

1.  **Vision-Anchored Docs**: Linking `README.md` to `vision/VISION.md` reinforces the core identity of the project, ensuring that all contributors (human or AI) are aligned with the foundational principles.
2.  **Self-Healing Context**: Documenting the Self-Healing mechanism within the documentation itself provides future agents with the instructions needed to utilize the diagnostic context (`PREVIOUS_FAILURE.log`) effectively.
