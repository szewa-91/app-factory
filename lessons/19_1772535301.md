# Lesson Learned: Resource-Aware Scheduling (Task 19)

## Technical Insight
To calculate accurate *current* CPU usage in Bash, `top -bn1` is often insufficient because the first iteration is frequently a system-wide average since the last reboot. Using `top -bn2 -d 0.2` and taking the second sample provides a much more accurate real-time snapshot. For RAM, parsing `free` output using `awk` to calculate the percentage `($3/$2)*100.0` is reliable across most modern Linux distributions.

## Implementation Pattern
Implementing a pre-check function in the scheduler that returns 1 on resource overload allows for graceful skipping of tasks without marking them as failures. This creates a natural "back-pressure" mechanism where agents pause activity until the server stabilizes, preventing cascaded failures or OOM kills during high load.

## Agent Safety
Headless agents should ensure they create and maintain their `agent.pid` file in the correct directory. If the scheduler resets an `IN_PROGRESS` task due to a missing PID file, it can lead to multiple agents running for the same task simultaneously (race condition), which we observed during this task execution. Always verify that the PID file location matches between the agent and the scheduler's check logic.
