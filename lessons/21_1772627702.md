# Lesson Learned: Task 21 — Nightly Backups

## Context
Implement `scripts/backup.sh` and a cron job for 3 AM daily backups of `factory.db` and `apps/`.

## Technical Insights

1. **Cron entry pre-existed**: The crontab already referenced `scripts/backup.sh` before the file was created. Always verify the crontab before adding a duplicate entry — in this case no crontab modification was needed.

2. **Selective tar exclusions are critical**: `apps/` contains `node_modules` and `.next` build caches that can balloon a 250 MB archive to gigabytes. Use `--exclude` for `node_modules`, `.next`, and `.git/objects/pack` to keep backups lean.

3. **`-C` flag for clean relative paths**: Using `tar -C "$WORKSPACE_DIR"` keeps archive members as `factory.db` and `apps/...` rather than absolute paths, making restoration portable.

4. **Retention via `find -mtime`**: `find ... -mtime +N -delete` is idiomatic for pruning old backups without maintaining a separate manifest. 14-day retention balances safety and disk usage.

5. **`set -euo pipefail` in backup scripts**: Ensures any partial failure (e.g., disk full mid-archive) exits non-zero, surfaces in `factory.log`, and won't silently produce a corrupt archive.
