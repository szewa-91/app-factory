# Lesson Learned: Task 21 retry #1 — Nightly Backups (Audit Fixes)

## Context
Audit Pass 2 identified four failures: broken tar exclusion patterns, world-readable archive permissions, missing git commit, and missing PLAN.md for this task.

## Technical Insights

1. **Tar `-C` and `--exclude` require relative patterns**: When using `tar -C /base/dir`, archive members are stored with paths relative to `/base/dir` (e.g. `apps/myapp/node_modules/`). `--exclude` patterns that include the absolute prefix (`/base/dir/apps/*/node_modules`) will never match those relative members. Always use relative patterns: `--exclude='apps/*/node_modules'`.

2. **Archive permissions need explicit hardening**: The default `umask` in cron environments may vary. Set `umask 077` before `tar` creation and follow with `chmod 600 "$ARCHIVE_PATH"` to guarantee `600` regardless of the calling environment.

3. **Verify exclusions with `tar -tzf | grep`**: The only reliable way to confirm exclusions worked is to inspect the resulting archive. A line count of `0` for `node_modules` is the acceptance criterion.

4. **Cron entry existence is independent of script correctness**: A pre-existing cron entry does not mean the script it invokes is correct. Always verify the script's behavior (exclusions, permissions, pruning) as a separate step.
