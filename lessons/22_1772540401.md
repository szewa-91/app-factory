# Lesson Learned: Integrating Testing Frameworks into Autonomous Application Factory

## Technical Insights

1. **Host-Based Testing for Dockerized Apps**: Running Playwright and Vitest directly on the host is often more efficient than inside production-ready containers (like `standalone` mode). This approach avoids bloating the production image with testing dependencies (browsers, `node_modules`).
2. **Playwright `webServer` for Verification**: Utilizing Playwright's `webServer` option allows for easy E2E testing against `npm run dev` on the host, which is independent of the Docker container. This serves as an excellent intermediate verification step before checking the actual container's responsiveness.
3. **Sequential Port Allocation and Testing**: In a factory environment where multiple apps might coexist, it's crucial that each verification run is sequential or uses isolated ports to avoid binding conflicts between testing processes (e.g., two apps both trying to use port 3000 for their dev server).
4. **Automating Test Infrastructure**: By automating the creation of `vitest.config.ts`, `playwright.config.ts`, and sample tests during bootstrapping, we ensure a consistent baseline of quality for all future projects. This lowers the friction for developers (and agents) to add meaningful tests later.
5. **Modern `package.json` Modification**: Using `npm pkg set` is a more robust way to update scripts programmatically than manual `sed` or `jq` operations on `package.json`.
