# Lesson Learned: Failure-Atomic Bootstrap + Trustworthy Verification

## Technical Insights
1. **Failure-atomic bootstrap prevents dead-end retries**: If app scaffolding or AI decomposition fails, rollback of both filesystem state and DB rows keeps reruns possible and avoids manual cleanup debt.
2. **AI-assisted design phases need deterministic fallback**: Agent capacity failures (quota/timeouts) should degrade to a local SPECS/tasks baseline so bootstrap remains operational and policy-compliant.
3. **Readiness-before-migration removes deploy race conditions**: Waiting for container internal health before `prisma migrate deploy` eliminates transient startup timing failures.
4. **TLS checks must preserve trust guarantees**: Public verification should never use `curl -k`; strict HTTPS validation catches broken certificate chains that would otherwise be silently accepted.
5. **Template-level port single-sourcing reduces drift**: Referencing `${APP_PORT}` across compose/deploy/verify scripts avoids hidden divergence when port policies evolve.
