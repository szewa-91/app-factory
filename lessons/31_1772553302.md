# Lesson Learned: Delimiter-Safe Scheduling + Build-Context Hygiene

## Technical Insights
1. **Non-whitespace separators prevent silent field shifts**: Parsing SQLite rows with tab delimiters can drop empty fields (`depends_on`) in Bash `read`, which misaligns downstream columns like `status` and `priority`; using a non-whitespace unit separator plus control-character sanitization keeps gating logic reliable.
2. **Resource checks need fallback-on-parse-failure, not just fallback-on-command-missing**: Even when `top`/`free` exist, output variance can break parsers; immediately falling back to `/proc/stat` and `/proc/meminfo` preserves scheduler back-pressure behavior across heterogeneous hosts.
3. **Template hardening should include generation-time integrity checks**: Bootstrap should fail fast if required templates are missing or unresolved tokens remain after rendering to avoid shipping broken deploy/verify scripts into new apps.
4. **Default `.dockerignore` is operational hardening**: Without it, generated app builds can send huge contexts (e.g., local `node_modules`) and create avoidable build latency/failure risk; seeding `.dockerignore` at bootstrap removes this class of instability.
5. **Runtime evidence catches gaps syntax checks miss**: A real bootstrap smoke run (`bootstrap -> deploy -> verify`) exposed context-size inefficiency and validated that source-level fixes hold under actual container startup and verification paths.
