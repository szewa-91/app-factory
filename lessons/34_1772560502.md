# Lesson Learned: Canonical Root Context Must Track Runtime Contracts

## Technical Insights
1. Scheduler prompt stability depends on root-document intent: if `PRD.md` drifts to a one-off initiative, autonomous infra tasks lose decision context and produce inconsistent changes.
2. Root specs for orchestration systems should encode executable contracts (schema, status lifecycle, API surface, verification entrypoints), not only architecture summaries.
3. Canonical docs are safest when anchored to runtime sources (`scripts/*.sh`, DB schema, route handlers) and validated with the same operational health check (`./verify.sh`) used by the scheduler pipeline.
