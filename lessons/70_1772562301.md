# Lesson Learned: TS DB Layer Parity Depends on Schema-First Typing

## Technical Insights
1. Mirror SQLite column names directly (`project_name`, `retry_count`, `depends_on`) in shared TypeScript interfaces to avoid accidental contract drift during Bash-to-TS migrations.
2. Use `COALESCE` in read/update statements for nullable numeric fields (for example `retry_count`) so strict TypeScript types can remain stable while honoring real SQLite nullability.
3. Parameterized statements must be enforced at the DB wrapper boundary for every dynamic value (task IDs, statuses, timestamps, project names) to prevent injection risks and keep scheduler behavior deterministic.
