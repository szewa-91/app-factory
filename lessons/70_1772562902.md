# Lesson Learned: Scheduler Scaffolds Still Need Runtime-Correct Behavior

## Technical Insights
1. Even during scaffold phases, entrypoints in production paths must execute real state transitions (for example claiming `READY` -> `IN_PROGRESS`) to satisfy anti-placeholder audit gates.
2. Keeping SQL statements fully static (no `${...}` interpolation) removes ambiguity in security reviews and prevents requirement drift when moving from Bash SQLite calls to `better-sqlite3` prepared statements.
3. DB mutation helpers should validate `run().changes` and throw on zero-row updates so scheduler retries and audits can detect missing-task conditions instead of silently succeeding.
