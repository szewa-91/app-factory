# Lesson Learned: Scheduler Parity Needs Explicit Data-Layer Contracts

Shell-parity scheduler rewrites become reliable when the database layer exposes exact transition primitives (plain status updates, timestamped risk transitions, and atomic claim updates) instead of generic CRUD helpers. Dependency checks are safer as a pure function that validates token format before querying each `dep_id`, which preserves bash behavior for malformed dependency strings. Resource gating is easiest to test when `/proc` parsing is isolated behind injectable file/sleep hooks, allowing deterministic CPU two-sample verification without real timing delays.
