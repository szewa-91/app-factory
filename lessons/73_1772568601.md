# Lesson Learned: Approval-Gate Logic Is Safest When Tested Through One Full Scheduler Cycle

For scheduler risk checks, the most reliable unit tests exercise `runSchedulerCycle` with a real temporary SQLite database while mocking only runtime boundaries (`checkResources`, agent execution, audit pipeline). This validates state transitions (`READY -> PENDING_APPROVAL` vs runnable paths) exactly as production uses them, avoids brittle internals-only assertions, and catches regressions in candidate selection + claim flow together.
