# Lesson Learned: Keep Lock-Contention Paths Observable During Scheduler Cutovers

When replacing a long-running shell scheduler with a compiled TypeScript entrypoint, lock-acquire failure paths must preserve production log semantics, not just exit codes. Explicitly logging lock contention (including the owner PID when available) in the entrypoint prevents silent cron no-ops and preserves operability during overlapping runs. Thin wrapper scripts should also fail fast with preflight checks for `node` and compiled artifacts so runtime drift is diagnosed immediately instead of surfacing as opaque scheduler failures.
