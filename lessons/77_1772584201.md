# Lesson Learned: Task 77 — Layout, Navigation & Global Styles

## Context
Task 77 implemented the root layout, sticky navigation, footer, and global CSS for personal-website (Next.js 16, Tailwind 4, App Router).

## Key Technical Insights

### Tailwind 4 `@theme` vs. standard config
Tailwind 4 drops `tailwind.config.js` in favor of `@theme inline { }` blocks in CSS. Custom design tokens (colors, fonts) live in `globals.css`, not a config file. CSS custom properties declared in `:root` can be referenced inside `@theme` with `var()`.

### Navigation must be `'use client'`
`usePathname()` and `useState` both require the Client Component directive. The rest of layout (layout.tsx, Footer.tsx) can stay as Server Components, so the client boundary is narrow.

### Forced dark mode via `className="dark"` on `<html>`
Setting `className="dark"` on the `<html>` element (not relying on `prefers-color-scheme`) gives deterministic dark theming for a portfolio site. Combined with `color-scheme: dark` in CSS, browser UI elements (scrollbars, inputs) also adopt dark styling.

### Conflict between task description and SPECS.md
The task description requested "Inter" font; SPECS.md specified "Geist Sans". SPECS.md is the authoritative design document and was already wired up — kept Geist Sans. Similarly, task asked for "Marcin Szewczyk | Senior Java Developer" as title — used that since it was explicit and both sources align on intent.

### `flex-col min-h-screen` on body for sticky footer
Using `flex flex-col min-h-screen` on `<body>` with `<main className="flex-1">` keeps the footer pinned to the bottom on short pages without JavaScript.
