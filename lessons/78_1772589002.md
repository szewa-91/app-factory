# Lesson Learned: Task 78 — Portfolio Section with og:image Fetching

## Context
Implemented `/portfolio` with a 3-column card grid, server-side og:image fetching, graceful fallback to gradient placeholders, hover animations, and staggered CSS entrance.

## Key Technical Insights

### Server-side og:image fetching in Next.js App Router
Using `fetch` with `next: { revalidate: 3600 }` in a server component causes Next.js to treat the page as ISR (revalidates every hour) rather than fully static. This is visible in the build output as `○ /portfolio  1h  1y`. Always add `AbortSignal.timeout(N)` to prevent hanging on slow/unreachable URLs.

### External image URLs and `<img>` vs `<Image>`
When og:image URLs come from unknown external domains, using `<Image>` from `next/image` requires enumerating all remote patterns in `next.config.ts`. Using a plain `<img>` tag with `eslint-disable-next-line @next/next/no-img-element` is pragmatic here since we don't control the source domains. Trade-off: no automatic optimization, but simpler and avoids config drift.

### CSS custom properties for staggered animation
Passing `--card-index` as a CSS custom property via inline `style={{ "--card-index": i } as React.CSSProperties}` and consuming it in a `.portfolio-card { animation-delay: calc(var(--card-index) * 100ms) }` rule avoids needing framer-motion for simple stagger effects. TypeScript requires the `as React.CSSProperties` cast for custom properties.

### og:image regex ordering
The meta tag `property` and `content` attributes can appear in either order in HTML. Always provide two regex variants to handle both orderings.
