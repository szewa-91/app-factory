# Lesson Learned: Task 79 — CV Section Implementation

## Context
Task 79 replaced placeholder CV data with the real Marcin Szewczyk CV and built a full-featured `/cv` page with timeline layout, skill badges, print support, and a print-to-PDF button.

## Key Technical Insights

### 1. Server + Client Component Split for Print
The CV page is a Server Component (no `'use client'`), but `window.print()` requires browser access. The solution: extract a tiny `PrintButton` client component (`'use client'`) and import it into the server page. This keeps the page statically generated while enabling the interactive print action.

### 2. Data Schema Evolution — Add Fields, Don't Break
The existing `Experience` interface had a single `description: string`. Switching to `highlights: string[]` allows multiple bullet points per job, which is critical for real CVs with multiple responsibilities. Always design data schemas to accommodate realistic content from the start.

### 3. `print:hidden` Utility Class in Tailwind v4
Tailwind v4 generates `print:hidden` via `@media print { display: none }`. The `PrintButton` uses `print:hidden` to hide itself during printing. The `globals.css` also adds `header` and `footer` to `display: none` for clean PDF output. Both mechanisms are necessary: Tailwind utility for component-level, global CSS for layout shell elements.

### 4. `print-color-adjust: exact` for Dark Backgrounds
Without `-webkit-print-color-adjust: exact` / `print-color-adjust: exact`, browsers strip background colors and borders in print mode, making dark-themed skill badges invisible. Always add this property when printing dark UIs.

### 5. Placeholder Data Debt
The initial stub in `data/cv.ts` had fake data (Assentics, Wroclaw University of Science and Technology) that differed from the actual CV. Always populate realistic or final data in data files early - fake placeholder data causes confusion and blocks audits that check for placeholder policy violations.
