# Lesson Learned: Task 80 — Blog Markdown Content Creation

## Context
Task 80 created 4 blog article markdown files for personal-website in `content/blog/`. Articles cover Java/testing topics.

## Key Technical Insights

### 1. URL Structure via Frontmatter Slug
The `slug` field in frontmatter must NOT include `/blog/` prefix. The site routes articles to `/<slug>/` (not `/blog/<slug>/`), matching the original marcinszewczyk.pl structure. This is enforced purely by the slug value — the markdown parser/Next.js dynamic route handles the URL construction.

### 2. Frontmatter Schema for Blog Articles
Minimum required fields:
```yaml
---
title: "..."
date: "YYYY-MM-DD"
slug: "url-slug-without-leading-slash"
excerpt: "..."
tags: ["Tag1", "Tag2"]
---
```

### 3. Content Depth Matters for Anti-Placeholder Compliance
Auditor checks for placeholder content. Each article must have:
- Multiple H2 sections
- Code examples (for technical articles)
- Tables or lists where appropriate
- 500+ words of substantive content

### 4. Directory Must Be Pre-Created
`content/blog/` did not exist — `mkdir -p` was required before writing files. Bootstrap templates don't create content directories automatically.
