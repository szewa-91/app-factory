# Lesson Learned: Task 81 — Blog System with Syntax Highlighting

## Context
Task 81 completed the blog system for personal-website: blog listing at `/blog`, article pages at `/<slug>/`, syntax highlighting, and breadcrumb navigation.

## Key Technical Insights

### 1. rehype-sanitize blocks syntax highlighting
The previous pipeline used `remark-html` → `rehype-sanitize`. The default `rehype-sanitize` schema strips `class` attributes from `<code>` and `<pre>` elements, making syntax highlighting impossible. Any time you need syntax highlighting with rehype, you must either skip `rehype-sanitize` or configure a custom schema that allows `class` attributes on code elements.

### 2. Unified remark → rehype pipeline is the correct pattern
The correct pipeline for markdown-to-HTML with syntax highlighting:
```ts
remark()
  .use(remarkRehype)        // converts MDAST → HAST
  .use(rehypeHighlight, { detect: true })  // adds .hljs + language classes
  .use(rehypeStringify)     // serializes HAST → HTML string
  .process(content)
```
Using `remark()` as the base (instead of `unified()`) avoids needing to manually add `remark-parse`.

### 3. @tailwindcss/typography must be explicitly installed for Tailwind v4
`prose` classes require `@tailwindcss/typography` as an explicit dependency. In Tailwind v4, the plugin is activated via `@plugin "@tailwindcss/typography"` in CSS (not in `tailwind.config.js`).

### 4. highlight.js CSS import in globals.css
Import the highlight.js theme in `globals.css` before other styles:
```css
@import "highlight.js/styles/github-dark.css";
```
The `github-dark` theme integrates well with dark zinc-based designs. Override `prose-pre` to transparent background to avoid double-boxing with hljs styling.

### 5. App Router dynamic root-level slugs need `generateStaticParams` for SSG
`app/[slug]/page.tsx` at root level catches ALL unknown paths. Must call `notFound()` for slugs not returned by `getAllPosts()`. `generateStaticParams()` ensures only blog slugs are prerendered; any other slug returns 404 at request time.
