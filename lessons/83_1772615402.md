# Lesson Learned: Task 83 — SEO, Sitemap & Metadata

## Context
Implemented full SEO for personal-website: `app/sitemap.ts`, `app/robots.ts`, root layout metadata with title template, per-page OpenGraph + Twitter cards, and SVG favicon.

## Technical Insights

1. **Next.js file-based sitemap/robots**: Exporting a default function from `app/sitemap.ts` and `app/robots.ts` is all that's needed — Next.js auto-generates `/sitemap.xml` and `/robots.txt` as static routes at build time.

2. **`metadataBase` is critical**: Setting `metadataBase: new URL("https://...")` in root layout metadata makes all relative `/og-image.png` references in child pages resolve to absolute URLs. Without it, OG image URLs would be relative and broken for social scrapers.

3. **Title template pattern**: Using `{ default: "...", template: "%s | Marcin Szewczyk" }` in the root layout means child pages only need `title: "Blog"` (not the full string), and the template appends the suffix automatically.

4. **Dynamic sitemap with getAllPosts()**: Calling `getAllPosts()` inside `sitemap()` means new blog posts are automatically included in the sitemap without any code changes — the blog lib reads from `content/blog/*.md`.

5. **SVG favicon support**: Modern browsers support SVG favicons via `<link rel="icon" type="image/svg+xml">`. Registering both SVG and `.ico` in `metadata.icons` provides a fallback for older browsers.

6. **OG image as static file**: For a static personal site, a pre-made `public/og-image.png` (1200×630) is the simplest approach. The file is referenced in metadata but needs to be created separately (e.g., via Figma/design tool).
