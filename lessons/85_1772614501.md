# Lesson Learned: Task 85 — Blog Post: Gra w programowanie

## Context
Added a new blog post with slug `gra-w-programowanie` to the personal website.

## Technical Insights

1. **Slug derives from filename**: The `lib/blog.ts` utility derives slug by stripping `.md` from the filename (`filename.replace(/\.md$/, "")`). Creating `gra-w-programowanie.md` automatically registers the post at `/gra-w-programowanie/` — no routing changes needed.

2. **Required frontmatter fields**: `title`, `date` (YYYY-MM-DD format), and `description` are mandatory. Missing any of these causes the post to be silently skipped with a console warning. The `tags` field is optional (defaults to `[]`).

3. **Static generation**: Posts are read at build time via `fs.readdirSync`. Adding a `.md` file and rebuilding (via `deploy.sh`) is sufficient to publish — no database, no API calls.

4. **Test stability**: Adding content files does not break existing tests. The test suite (`vitest run`, 14 tests) remained green after adding the new post.
