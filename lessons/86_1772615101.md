# Lesson Learned: Task 86 — Blog Post: Problem N+1

## Context
Added a new blog post with slug `problem-n-plus-1` to the personal website.

## Technical Insights

1. **Slug derives from filename**: `lib/blog.ts` uses `filename.replace(/\.md$/, "")` as the slug — the frontmatter `slug` field is not used for routing, only the filename matters. Always name the file to match the desired URL slug.

2. **Required frontmatter**: `title`, `date` (YYYY-MM-DD), and `description` are mandatory. Missing any of them causes the post to be silently skipped. `tags` is optional but recommended.

3. **No code changes needed for new posts**: The filesystem-first blog architecture means adding a `.md` file to `content/blog/` is sufficient — no route registration, no CMS update, no import needed.

4. **Test suite is stable**: The 14 existing vitest tests are unaffected by new blog posts (they test rendering, not content), so adding posts never breaks tests.
